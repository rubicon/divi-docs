# 03-assets — What Goes Here

Working assets for the project. Populated as the project progresses.

## Subfolders (create as needed)

- **`copy/`** — Homepage copy, page drafts, FAQ content, blog posts. Version drafts (`homepage-v1.md`, `homepage-v2.md`, etc.) rather than overwriting.
- **`photos/`** — Professional headshots, office photos, team photos. Original high-res files go here. Web-optimized versions belong in the build folder once we're in build phase.
- **`brand/`** — Logo files, color tokens, typography choices, any existing brand materials from the client.

## Naming Conventions

- Lowercase, hyphenated filenames: `homepage-hero-v2.md`, not `Homepage Hero V2.md`
- Drafts get versioned: `-v1`, `-v2`, `-final`. When something is truly final, update and move to `02-deliverables/` if it's going to the client.
- Dates in filenames use ISO format: `YYYY-MM-DD-short-slug.md` (e.g. `2026-04-16-kickoff-draft.md`)
