Append a new entry to `01-context/activity-log.md` documenting what happened in this work session.

Before writing, ask the user (if you don't already know):
- What surface are you working on — Cowork, Claude Code, or Claude.ai chat?
- What did this session accomplish? (brief bullets)
- Is anything in progress / partial?
- Is anything queued for the next session?

Format the entry per the template at the top of `activity-log.md`:

```
### YYYY-MM-DD — [Surface] — [Short topic]
- [What you did — one sentence per bullet]
- **In progress:** [partial work, or "none"]
- **Queued:** [what next session should pick up, or "none"]
```

Keep it short. The activity log gets read at the start of every future session — bloat kills the habit.

Arguments: $ARGUMENTS
